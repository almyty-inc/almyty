# Agents

An agent takes an input, does work with one or more LLMs and tools, and returns
a result. almyty has two kinds:

- **[Autonomous](/agents/autonomous)** — you give it instructions and tools, and
  the model decides what to do. Several models can collaborate in one agent, the
  output is verified before it returns, and good runs become reusable skills.
  This is the default for open-ended work.
- **[Workflow](/agents/builder)** — you draw the steps as a graph. Use it when a
  task must run the same way every time.

![Agents list](/screenshots/agents-list.png)

## Modes

| Mode | Best for | How it works |
|------|----------|--------------|
| **Autonomous** | Open-ended reasoning | Instructions, tools, memory, and a verifier panel; the model drives. See [Autonomous agents](/agents/autonomous). |
| **Workflow** | Deterministic pipelines | A visual DAG of typed nodes connected by edges. See the [Builder](/agents/builder). |

Both modes share the same building blocks underneath — tools, LLM providers,
memory, verification, scheduling, and the OpenAI-compatible endpoint — so you can
pick per agent without giving anything up.

## Create an agent

In the UI: **Agents → Create Agent**, then pick **Autonomous** or **Workflow**.

- **Autonomous** — enter instructions, attach tools, enable memory, add
  constraints, and configure the verifier panel. Full detail in
  [Autonomous agents](/agents/autonomous).
- **Workflow** — the canvas opens with a default Input → LLM → Output pipeline.
  Drag nodes from the palette, connect them, and configure each in the side
  panel. Full detail in the [Builder](/agents/builder).

From the list you can activate or deactivate an agent, duplicate it, import one
from JSON, or delete it.

### Via the API

```bash
# Autonomous
curl -X POST https://api.almyty.com/agents \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Support Agent",
    "mode": "autonomous",
    "instructions": "Answer support questions using the knowledge-base tool.",
    "toolIds": ["tool-uuid-1", "tool-uuid-2"],
    "memoryEnabled": true
  }'

# Workflow (abbreviated — see the Builder for the full pipeline shape)
curl -X POST https://api.almyty.com/agents \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "name": "Research Agent", "mode": "workflow", "pipeline": { "nodes": [], "edges": [] } }'
```

### Invoke an agent

Every agent, whichever mode, answers on the same endpoint:

```bash
curl -X POST https://api.almyty.com/agents/{id}/invoke \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "input": { "query": "What is the weather in Berlin?" } }'
```

Agents are also reachable through the
[OpenAI-compatible API](/api-reference/openai-compatible), so an existing OpenAI
SDK can call them by changing the base URL.

## Lifecycle

| Status | Description |
|--------|-------------|
| `draft` | Being built, not yet invokable |
| `active` | Live and accepting invocations |
| `inactive` | Paused; invocations are rejected |
| `error` | Has a configuration issue |

## Execution model

An **autonomous** agent sends the input to the primary model with its
instructions and tools, lets the model call tools and reason, runs the
[verifier panel](/agents/autonomous#verification) over the result, and revises if
a checker fails.

A **workflow** agent validates the input against the Input node's schema,
topologically sorts the pipeline, executes each node in dependency order, resolves
[template expressions](/agents/template-expressions) at runtime, and returns the
Output node's result.
