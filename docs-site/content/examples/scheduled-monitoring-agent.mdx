# A scheduled agent that watches and reports

**Goal:** every morning at 8:00, an agent checks last night's failed payments, groups them by
reason, and drops a summary in your team's Slack — before anyone has opened a dashboard.

## 1. Build the agent

Create an autonomous agent, attach the tools it needs — your payments API, a Slack tool — and
give it the standing instruction: *pull failed charges since midnight, group by decline
reason, and post a short digest; if none, say so.* Turn on **memory** so it can note
recurring failures across days.

## 2. Put it on a schedule

On the agent's **Schedule** panel, set an interval — every day, or every few hours for
tighter loops. almyty triggers the run on that cadence with no external cron; a missed
window doesn't pile up.

![The schedule configuration for an agent](/screenshots/schedule-config.png)

## 3. Push the result out

You have two ways to deliver. The agent can call a Slack (or email) tool directly as its last
step, or you can register an **outbound webhook** so almyty POSTs the run result to an
endpoint you own — a Slack incoming webhook, an internal service — on `agent.execution.completed`.

![Configuring an outbound completion webhook](/screenshots/webhook-config.png)

## 4. Let it run

From then on the digest arrives on schedule. Because the run is a normal agent run, it's in
the audit log, it respects the same constraints and budgets, and you can open any past run to
see exactly what it queried and sent.

The same pattern covers uptime checks, nightly report generation, SLA-breach alerts, and
data-freshness monitors — anything that's "look at X on a cadence and tell someone."
