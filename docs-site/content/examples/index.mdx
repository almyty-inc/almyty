# Examples

Buildable, real-world recipes. Each takes a concrete goal and walks through the almyty pieces
that get you there — the same core primitives (tools, agents, gateways, memory, the runner)
composed different ways.

## Developers & integration

- [Give Claude Code your own API](/examples/give-claude-your-api) — import a schema, publish an MCP gateway, call it from Claude.
- [Turn an npm package into tools](/examples/npm-package-tools) — point almyty at `stripe` and get typed tools, no wrappers.
- [Modernize a legacy SOAP service](/examples/modernize-legacy-soap) — wrap a 2006 WSDL service as modern MCP tools without touching it.

## Autonomous agents

- [A support agent that checks its own work](/examples/self-checking-support-agent) — cross-vendor verification + human approval for refunds.
- [Multi-model debate for high-stakes calls](/examples/multi-model-debate) — several models argue, a judge decides.
- [An agent that remembers across runs](/examples/agent-that-remembers) — memory tiers so knowledge accumulates instead of resetting.
- [Promote a good run into a skill](/examples/promote-run-to-skill) — turn one solved task into a reusable Agent Skill.
- [A scheduled agent that watches and reports](/examples/scheduled-monitoring-agent) — run on a cadence, push the result to Slack.

## Runner

- [Give an agent a machine to work on](/examples/expose-any-machine) — expose any machine — laptop, build server, VPC box — for an agent to execute on.
- [Orchestrate a multi-vendor coding fleet](/examples/multi-vendor-coding-fleet) — one agent driving Claude Code, Codex, and aider on your repo.

## Enterprise & governance

- [Put a governed agent into production](/examples/governed-production-agent) — constraints, verify gates, approvals, and audit.
- [Air-gapped, self-hosted agents](/examples/air-gapped-self-host) — open-source models, nothing leaves your network.
