# Examples

Real jobs, built end to end with almyty. Each recipe starts from the outcome, names its
stack (trigger, models, tools, mode), and walks the concrete steps. Find the one closest
to what you want, follow it, and swap in your own APIs and channels.

![Agents built from these recipes, running in the dashboard](/screenshots/agents-list.png)

Every recipe is assembled from the same pieces: [tools](/tools) (any API or npm package
you connect), [gateways](/gateways) (how agents and clients reach them),
[agents](/agents/autonomous), [memory](/memory), and the [runner](/runner). A name like
"CRM API" means an API you connect, not a built-in connector.

## Give your AI access to APIs and data

Turn the systems you already run into tools an agent can call.

- **[Give Claude Code your own API](/examples/give-claude-your-api)**: import a schema, publish an MCP gateway, call it from Claude in one line.
- **[Modernize a legacy SOAP service](/examples/modernize-legacy-soap)**: wrap a WSDL as modern MCP tools, source untouched.
- **[Turn an npm package into tools](/examples/npm-package-tools)**: point almyty at a package like `stripe` and get typed tools.
- **[Connect Google Drive and SharePoint](/examples/connect-google-sharepoint)**: search and summarize your docs, then write back.
- **[Query your database in plain English](/examples/nl-query-your-database)**: the agent writes and runs read-only SQL, and shows the query it used.
- **[Give an agent a machine to work on](/examples/expose-any-machine)**: expose a laptop, build server, or VPC box through the runner.

## Automate real work

Put an agent on a recurring job, a channel, or a webhook and let it do the task.

- **[Triage support tickets and draft replies](/examples/support-ticket-triage)**: looks up the order, drafts a reply, escalates refunds to a human.
- **[Deflect L1 questions from your docs](/examples/docs-deflection-bot)**: answers in-channel with a cited page, opens a ticket only when it cannot.
- **[Enrich and route inbound leads](/examples/lead-enrichment-routing)**: scores against your ICP, writes back to the CRM, pings the right rep.
- **[Turn call notes into CRM updates](/examples/meeting-notes-to-crm)**: extracts next steps, updates the opportunity, drafts a follow-up.
- **[Triage and label GitHub issues](/examples/github-issue-triage)**: labels, links duplicates, routes, and requests repros.
- **[Review pull requests](/examples/pr-review-bot)**: flags bugs, missing tests, and policy violations inline, and never approves.
- **[Summarize on-call alerts](/examples/oncall-alert-triage)**: gathers metrics and logs, correlates with past incidents, posts a summary.
- **[Run a multi-vendor coding fleet](/examples/multi-vendor-coding-fleet)**: one planner driving Claude Code, Codex, and aider on a single repo.
- **[Post a nightly data-quality digest](/examples/nightly-data-quality-digest)**: checks freshness and anomalies on a schedule, reports only what is wrong.
- **[Reconcile Stripe payouts](/examples/stripe-payout-reconciliation)**: matches Stripe against the ledger daily and reports only the exceptions.
- **[Run a cross-vendor contract review](/examples/cross-vendor-contract-review)**: several models argue the risks, a judge writes the memo.

## Govern and secure agents

Add constraints, budgets, approvals, and access control before an agent touches production.

- **[Run a governed refund agent](/examples/governed-refund-agent)**: constraints, cross-vendor verify, human approval, and a full audit trail.
- **[Govern AI spend across teams](/examples/govern-ai-spend)**: per-team budgets with soft and hard caps, enforced at run start.
- **[Enforce approval policies org-wide](/examples/approval-policies)**: map action plus threshold to a required approver across every agent.
- **[Role-based access for a multi-team org](/examples/multi-team-rbac)**: teams, per-resource roles, least privilege by default.
- **[Onboard with SSO and SCIM](/examples/sso-scim-onboarding)**: SAML or OIDC sign-in, SCIM provisioning, groups mapped to roles.
- **[Stream your audit trail to a SIEM](/examples/audit-to-siem)**: every action to Splunk or Datadog, searchable and alertable.

## Self-host and comply

Keep data in your network and meet regulatory requirements.

- **[Air-gapped, self-hosted agents](/examples/air-gapped-self-host)**: local models, data residency, nothing leaves your network.
- **[Ship EU AI Act-compliant agents](/examples/eu-ai-act-compliance)**: AI disclosure on every channel plus an Annex IV technical-doc export.
