# An agent that remembers across runs

**Goal:** a research assistant that gets better the more you use it — it should recall the
accounts it has already dug into, the conventions your team uses, and the conclusions it
reached last week, instead of starting cold every session.

## How memory is organized

Memory lives in a dedicated **Memory** workspace — a canonical, bi-temporal store with
multi-backend routing. Every entry carries a **tier** that sets its lifetime and scope:

| Tier | Holds | Lives |
| --- | --- | --- |
| Short | The current run's working context | The run |
| Project | Facts scoped to one project or agent | Until changed |
| Long | Durable, consolidated knowledge | Across runs |
| Shared | Knowledge several agents can read | Org-wide |

Agents write to these tiers as they work and read from them on the next run, so knowledge
accumulates. Durable facts — a customer's SLA, a lesson from a past analysis — land in the
**long** tier and persist.

![Durable long-tier memories, each with its tier, labels, and embedding status](/screenshots/memory.png)

## 1. Enable it on the agent

Turn on memory for the agent and choose a backend — memory persists in your own store
(Postgres, or one of several supported backends), so what the agent learns stays with you.
Filter the workspace by tier to see just the short-term working set or the durable knowledge.

![Filtering the Browse view by tier — a short-term working memory](/screenshots/memory-write.png)

## 2. Let it write as it learns

During a run the agent records what's worth keeping — "Acme's fiscal year starts in July,"
"the EU dataset lags two days." Entries are bi-temporal: almyty tracks both when a fact was
true and when it was recorded, so you can ask what the agent knew at a past point in time,
not only what it knows now.

## 3. Retrieve on the next run

On the following run the agent pulls the relevant memories — by semantic similarity and
recency — back into context. Ask it to "continue the Acme analysis" and it already has the
fiscal calendar and the caveats from last time.

## 4. Share across a team of agents

Promote durable facts to the **shared** tier and other agents in the org read them too — your
triage agent and your reporting agent work from the same institutional knowledge, and it
stays in sync across them.
