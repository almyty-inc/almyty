# 🔌 Give Claude Code your own API

**Stack** — `Trigger: Claude Code / any MCP client` · `Tools: your API (any schema)` · `Gateway: MCP (+ A2A, UTCP)` · `Auth: x-api-key` · `Output: Claude calls your API`

**Who it's for** — anyone who wants Claude Code, Cursor, or Claude Desktop to call an API they already run.
**What it does** — turns your API into typed tools and publishes them as an MCP gateway, so an MCP client can call your service directly — with auth, rate limits, and audit handled by almyty, and no tool wrappers to write.

## Build it

1. **Connect the API.** In *APIs → Connect API*, paste a schema URL — OpenAPI, GraphQL, SOAP/WSDL, or Protobuf. almyty generates a typed tool for every operation.

   ![An imported API — operations turned into tools](/screenshots/quickstart-api-imported.png)

2. **Publish an MCP gateway.** Group the tools you want to expose and publish them as an **MCP** gateway. Its endpoint is `https://api.almyty.com/{org}/{gateway}`, and the *Authentication* panel generates an `x-api-key`.

   ![The gateway with its endpoint and key auth](/screenshots/quickstart-gateway-mcp.png)

3. **Add it to your client.** *Quick Setup* gives you a copy-paste command:
   ```bash
   claude mcp add petstore --transport http https://api.almyty.com/acme/petstore
   # clients send: x-api-key: <your gateway key>
   ```
4. **Use it.** Ask Claude something that needs your API — it calls the generated tools through the gateway:
   ```text
   > look up pet #2 and show its status
     Claude calls get_pet_by_id via almyty → "Rosie", available
   ```

## Make it yours

- The same gateway also speaks **A2A** and **UTCP**, so other agents and clients reach identical tools.
- Scope which operations each key can call; rotate keys from the Authentication panel.
- Front a [legacy SOAP service](/examples/modernize-legacy-soap) the same way.

**Uses** — `Platform` · `MCP` · `Claude Code`
